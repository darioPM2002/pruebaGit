Flujos de trabajo github

Propuestas personales para trabajar individual y colaborativamente en github. GitHub es una plataforma de desarrollo colaborativo basada en la web que utiliza Git, un sistema de control de versiones distribuido.

Flujo básico

Este flujo es el básico, excelente para un repositorio individual

    1

    Crear un repositorio

    En caso de que sólo necesites un repositorio local deberás correr git init, para crear un repositorio deberás ingresar a github.com, y en la opción de crear un nuevo repositorio debes crear uno configurandolo con las necesidades del proyecto (hacerlo privado, tipo de licencia, agregar colaboradores etc.).

    2

    Clonar un repositorio

    Clonar un repositorio nos permite bajar un repositorio remoto a un repositorio local, con el fin de trabajar en una copia de él. para esto ncesitas tener el ssh del repositorio el cual encuentras en la página de github del repositorio, con el ssh, en tu computadora debes alojarte en algún directorio en la terminal donde quieras bajar el repositorio remoto con esto debes de ejecutar git clone <ssh del repositorio>, con esto tendras el repositorio remoto de forma local.

    3

    Agregar cambios (Staging area)

    Cuando hagas algún cambio en tu repositorio local y quieras guardar los cambios debes ejecutar git add <nombre del archivo> o git add -A el primero para salvar un archivo en específico y el segundo para salvar todos los archivos que hayas modificado, esto hace que los archvios pasen el staging area, es decir los archvivos que se agregarán en el próximo commit.

    4

    Guardar cambios

    Ya con los cambios agregados debes guardar esos cambios y generar historial de versiones, para esto se ejecuta git commit -m "Mensaje descriptivo del cambio", el nombre del cambio se verá para tener un control de qué pasó en esa versión.

    5

    Subir los cambios al repositorio remoto

    Una vez que tus cambios están listos y actualizados con la rama principal remota, sube tu rama al repositorio remoto, la rama por defecto es main, pero puedes usar cualquiera que esté disponible dependiendo del caso el comando es git push origin nombre-de-la-rama

    6

    IMPORTANTE

    Hay dos comnandos los cuáles se deben ejecutar dependiendo cada programador pero son de suma importancia, no entrar dentro del woekflow anterior debido a que es comun ejecutarlos entre cada paso, los comandos son:

    git pull origin "nombre de la rama" bajas todos los cambios del repositorio remoto al local, es decir realizas actualizaciones al último estado

    git status te muestra toda la información del repositorio, como los archivos en el staging areea
